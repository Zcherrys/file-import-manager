# File Import Manager - Obsidian 插件

这个插件可以接管 Obsidian 的文件导入功能，当您将文件拖入 Obsidian 时，插件会根据文件类型自动将文件复制到指定的文件夹中，而不经过 Obsidian 的内部导入流程。

## 功能特点

- 接管 Obsidian 的文件拖放导入功能
- 根据文件类型自动将文件复制到不同的文件夹
- 支持自定义文件类型与目标文件夹的映射关系
- 提供默认文件夹配置，用于处理未指定类型的文件
- 包含启用/禁用插件的开关
- 可以自动重命名自定义导入文件名称
- 可以自定义附件后缀名分类存放

## 安装方法

1. 确保您已经安装了 Node.js 和 npm
2. 克隆或下载此仓库到您的计算机
3. 在插件目录中运行以下命令安装依赖：
   ```bash
   npm install
   ```
4. 构建插件：
   ```bash
   npm run build
   ```
5. 将插件添加到 Obsidian：
   - 打开 Obsidian 设置
   - 点击 "第三方插件" -> "已安装插件" -> "浏览"
   - 打开 "开发模式"
   - 点击 "从文件系统安装插件"
   - 选择插件目录

## 使用方法

1. 在 Obsidian 设置中找到 "File Import Manager" 设置
2. 确保插件已启用
3. 配置默认文件夹路径和文件类型映射
4. 直接将文件拖入 Obsidian 窗口，插件会自动处理文件导入

## 设置选项

### 启用/禁用插件
- 通过切换开关可以随时启用或禁用插件
- 禁用后，Obsidian 将使用默认的文件导入行为

### 默认文件夹
- 当拖入的文件类型没有特定映射时，将使用此文件夹作为目标位置
- 默认值为 `./imports`

### 文件类型映射
- 您可以为不同类型的文件设置不同的目标文件夹
- 默认配置包括常见的图片、文档、音频、视频等文件类型
- 您可以添加、编辑或删除任何文件类型映射

## 文件类型映射配置示例

默认的文件类型映射配置如下：

- `.jpg`, `.jpeg`, `.png`, `.gif`, `.svg` -> `./images`
- `.pdf`, `.doc`, `.docx`, `.xls`, `.xlsx`, `.ppt`, `.pptx` -> `./documents`
- `.md`, `.txt` -> `./notes`
- `.mp3`, `.wav` -> `./audio`
- `.mp4`, `.mov` -> `./video`

## 注意事项

- 确保目标文件夹路径是相对于您的 Obsidian 库的有效路径
- 插件会自动创建不存在的目标文件夹
- 如果目标位置已存在同名文件，插件会在文件名后添加数字后缀以避免覆盖
- 由于接管了系统的导入，会导致系统设置里的**附件相关设置**不可用
- 默认【使用wiki链接】
- 默认双链形式【基于仓库根目录的绝对路径】

## 开发说明

如果您想为此插件贡献代码或进行修改，请遵循以下步骤：

1. 安装依赖：`npm install`
2. 开发模式（自动重新编译）：`npm run dev`
3. 构建生产版本：`npm run build`
